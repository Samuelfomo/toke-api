Système de Gestion des Licences toké

Document technique d’implémentation

📋 Vue d’Ensemble Architecturale

Base de Données Centrale (master)

  • toke_master : Base centralisée pour la gestion des licences multi-pays
  • toke_tenant_XXX : Bases isolées par tenant pour les données opérationnelles (pointages, sites, etc.)

Principe Multi-Pays

  • Infrastructure centralisée : Un seul serveur pour tous les pays
  • Devise de référence : USD avec conversion automatique vers devises locales
  • Taxation : Application automatique selon le pays et statut TVA du tenant
  • Langue : Choix du tenant lors de l’inscription

🌍 Tables de Référence Internationales

COUNTRY (Pays Supportés)

Objectif : Centraliser les paramètres spécifiques à chaque pays (devise, langue par défaut, règles fiscales)

CREATE TABLE country (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code CHAR(2) UNIQUE NOT NULL, -- ISO 3166-1 (CM, FR, US, etc.)
    name_en VARCHAR(100) NOT NULL, -- Nom en anglais
    name_local VARCHAR(100), -- Nom dans langue locale
    default_currency_code CHAR(3) NOT NULL, -- XAF, EUR, USD
    default_language_code CHAR(2) NOT NULL, -- fr, en, es
    is_active BOOLEAN DEFAULT TRUE,
    timezone_default VARCHAR(50) NOT NULL, -- Africa/Douala, Europe/Paris
    phone_prefix VARCHAR(10), -- +237, +33, +1
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CURRENCY (Devises et Taux de Change)

Objectif : Gérer les conversions USD vers devises locales avec historique pour cohérence des factures

CREATE TABLE currency (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code CHAR(3) UNIQUE NOT NULL, -- XAF, EUR, USD
    name VARCHAR(50) NOT NULL, -- Franc CFA, Euro, Dollar
    symbol VARCHAR(10) NOT NULL, -- FCFA, €, $
    decimal_places SMALLINT DEFAULT 2, -- Nombre de décimales (2 pour EUR, 0 pour JPY)
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE exchange_rate (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    from_currency_code CHAR(3) NOT NULL, -- Toujours USD (devise de référence)
    to_currency_code CHAR(3) NOT NULL,
    exchange_rate DECIMAL(12,6) NOT NULL, -- 1 USD = 700.000000 XAF
    effective_date TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    is_current BOOLEAN DEFAULT TRUE, -- Seul le taux courant est TRUE
    created_by UUID, -- Qui a mis à jour le taux
    created_at TIMESTAMPTZ DEFAULT NOW(),
    
    CONSTRAINT valid_exchange_rate CHECK (exchange_rate > 0),
    CONSTRAINT fk_from_currency FOREIGN KEY (from_currency_code) REFERENCES currency(currency_code),
    CONSTRAINT fk_to_currency FOREIGN KEY (to_currency_code) REFERENCES currency(currency_code)
);

CREATE TABLE tax_rule (

GLOBAL_LICENSE (Licence Globale Multi-Devises)

Objectif : Gérer les licences avec calculs automatiques multi-devises et taxes

CREATE TABLE global_license (
    global_license_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    
    -- Configuration licence
    license_type license_type_enum NOT NULL DEFAULT 'cloud_flex',
    billing_cycle_months SMALLINT NOT NULL CHECK (billing_cycle_months IN (1,3,6,12)),
    
    -- Pricing (toujours en USD de base)
    base_price_usd DECIMAL(10,2) NOT NULL DEFAULT 3.00, -- 3$ par employé par mois
    minimum_seats SMALLINT NOT NULL DEFAULT 5,
    
    -- Période et renouvellement
    current_period_start TIMESTAMPTZ NOT NULL,
    current_period_end TIMESTAMPTZ NOT NULL,
    next_renewal_date TIMESTAMPTZ NOT NULL,
    
    -- Calculs automatiques
    total_seats_purchased INTEGER GENERATED ALWAYS AS (
        COALESCE((SELECT COUNT(*) FROM employee_license WHERE global_license_id = global_license.global_license_id), 0)
    ) STORED,
    
    -- Statut
    license_status license_status_enum NOT NULL DEFAULT 'active',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    
    -- Contraintes
    CONSTRAINT valid_period_dates CHECK (current_period_end > current_period_start),
    CONSTRAINT valid_renewal_date CHECK (next_renewal_date >= current_period_end),
    CONSTRAINT fk_license_tenant FOREIGN KEY (tenant_id) REFERENCES tenant(tenant_id) ON DELETE CASCADE
);

EMPLOYEE_LICENSE (Licences Employés avec Calcul Anti-Fraude)

Objectif : Suivi individuel des licences employés avec statut de facturation calculé automatiquement selon les règles anti-fraude

CREATE TABLE employee_license (
    employee_license_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    global_license_id UUID NOT NULL,
    employee_id UUID NOT NULL, -- Référence vers toke_tenant_xxx.utilisateur
    employee_code VARCHAR(50) NOT NULL,
    
    -- Dates importantes
    activation_date TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deactivation_date TIMESTAMPTZ NULL,
    last_activity_date TIMESTAMPTZ NULL, -- Dernière fois qu'il a pointé
    
    -- Statuts
    contractual_status contractual_status_enum NOT NULL DEFAULT 'active',
    declared_long_leave BOOLEAN NOT NULL DEFAULT FALSE,
    long_leave_declared_by UUID NULL,
    long_leave_declared_at TIMESTAMPTZ NULL,
    long_leave_type leave_type_enum NULL,
    long_leave_reason TEXT NULL,
    
    -- Statut de facturation calculé automatiquement
    computed_billing_status billing_status_computed_enum GENERATED ALWAYS AS (
        CASE
            -- Règle 1: A pointé dans les 7 derniers jours = TOUJOURS facturable
            WHEN last_activity_date >= NOW() - INTERVAL '7 days' THEN 'billable'::billing_status_computed_enum
            
            -- Règle 2: Pas de pointage + déclaré en congé = non facturable
            WHEN last_activity_date < NOW() - INTERVAL '7 days' 
                 AND declared_long_leave = TRUE THEN 'non_billable'::billing_status_computed_enum
            
            -- Règle 3: Pas de pointage + pas déclaré + période de grâce
            WHEN last_activity_date < NOW() - INTERVAL '7 days'
                 AND declared_long_leave = FALSE
                 AND (deactivation_date IS NULL OR deactivation_date >= NOW() - INTERVAL '7 days') 
                 THEN 'grace_period'::billing_status_computed_enum
            
            -- Règle 4: Autres cas = non facturable
            ELSE 'non_billable'::billing_status_computed_enum
        END
    ) STORED,
    
    -- Période de grâce
    grace_period_start TIMESTAMPTZ NULL,
    grace_period_end TIMESTAMPTZ NULL,
    
    -- Métadonnées
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    
    -- Contraintes anti-fraude
    CONSTRAINT no_long_leave_with_recent_activity CHECK (
        NOT (declared_long_leave = TRUE AND last_activity_date >= NOW() - INTERVAL '7 days')
    ),
    CONSTRAINT valid_long_leave_data CHECK (
        (declared_long_leave = FALSE) OR 
        (declared_long_leave = TRUE AND long_leave_declared_by IS NOT NULL AND long_leave_declared_at IS NOT NULL)
    ),
    CONSTRAINT valid_deactivation_date CHECK (
        deactivation_date IS NULL OR deactivation_date >= activation_date
    ),
    
    CONSTRAINT fk_employee_license FOREIGN KEY (global_license_id) REFERENCES global_license(global_license_id) ON DELETE CASCADE
);

💰 Tables de Facturation et Paiements

BILLING_CYCLE (Cycles de Facturation Multi-Devises)

Objectif : Gérer les cycles de facturation avec calculs automatiques en devises locales et taxes

CREATE TABLE billing_cycle (
    billing_cycle_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    global_license_id UUID NOT NULL,
    
    -- Période
    period_start TIMESTAMPTZ NOT NULL,
    period_end TIMESTAMPTZ NOT NULL,
    
    -- Comptage employés
    base_employee_count INTEGER NOT NULL, -- Nombre au début de période
    final_employee_count INTEGER NOT NULL, -- Nombre facturé (calculé)
    
    -- Montants en USD (référence)
    base_amount_usd DECIMAL(12,2) NOT NULL,
    adjustments_amount_usd DECIMAL(12,2) DEFAULT 0,
    subtotal_usd DECIMAL(12,2) NOT NULL,
    tax_amount_usd DECIMAL(12,2) DEFAULT 0,
    total_amount_usd DECIMAL(12,2) NOT NULL,
    
    -- Montants en devise locale (pour affichage client)
    billing_currency_code CHAR(3) NOT NULL,
    exchange_rate_used DECIMAL(12,6) NOT NULL,
    base_amount_local DECIMAL(12,2) NOT NULL,
    adjustments_amount_local DECIMAL(12,2) DEFAULT 0,
    subtotal_local DECIMAL(12,2) NOT NULL,
    tax_amount_local DECIMAL(12,2) DEFAULT 0,
    total_amount_local DECIMAL(12,2) NOT NULL,
    
    -- Taxes appliquées
    tax_rules_applied JSONB, -- Détail des taxes appliquées
    
    -- Statut et dates
    billing_status billing_status_enum NOT NULL DEFAULT 'pending',
    invoice_generated_at TIMESTAMPTZ,
    payment_due_date TIMESTAMPTZ NOT NULL,
    payment_completed_at TIMESTAMPTZ,
    
    -- Métadonnées
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    
    CONSTRAINT fk_billing_license FOREIGN KEY (global_license_id) REFERENCES global_license(global_license_id) ON DELETE CASCADE,
    CONSTRAINT fk_billing_currency FOREIGN KEY (billing_currency_code) REFERENCES currency(currency_code),
    CONSTRAINT valid_amounts CHECK (total_amount_usd = subtotal_usd + tax_amount_usd)
);

⚙️ Triggers PostgreSQL

1. Trigger de Mise à Jour des Timestamps

Objectif : Maintenir automatiquement les colonnes updated_at

Description fonctionnelle :

  • Se déclenche à chaque UPDATE sur toutes les tables principales
  • Met automatiquement updated_at = NOW()
  • Appliqué sur : tenant, global_license, employee_license, billing_cycle, etc.

2. Trigger de Détection de Fraude - Congés Massifs

Objectif : Détecter automatiquement les patterns suspects de mise en congé

Description fonctionnelle :

  • Se déclenche lors de modification de declared_long_leave ou long_leave_type
  • Calcule le pourcentage d’employés en congé technique pour le tenant
  • Si > 15% : Insère une alerte dans fraud_detection_log avec niveau “medium”
  • Si > 30% : Niveau “high” et notification d’urgence

3. Trigger de Validation Anti-Fraude

Objectif : Empêcher les incohérences de données au niveau base

Description fonctionnelle :

  • Se déclenche avant INSERT/UPDATE sur employee_license
  • Vérifie qu’un employé avec pointage récent ne peut pas être mis en congé long
  • Bloque l’opération et retourne une erreur explicite si violation
  • Garantit l’intégrité de la règle “pointage récent = facturable”

4. Trigger de Calcul Automatique des Montants

Objectif : Recalculer automatiquement les montants lors de changements

Description fonctionnelle :

  • Se déclenche lors de création/modification d’un billing_cycle
  • Applique automatiquement les taux de change courants
  • Calcule les taxes selon les règles du pays du tenant
  • Met à jour les montants locaux en conséquence

5. Trigger de Synchronisation d’Activité

Objectif : Maintenir à jour les données d’activité des employés

Description fonctionnelle :

  • Se déclenche lors de mise à jour de last_activity_date
  • Met à jour automatiquement la table activity_monitoring
  • Recalcule les compteurs de pointage (7 jours, 30 jours)
  • Détecte les périodes d’absence suspectes

📊 Fonctions d’Agrégation PostgreSQL

1. calculate_billable_employees(license_uuid)

Objectif : Compter les employés facturables pour une licence donnée

Description :

  • Paramètre : UUID de la licence globale
  • Retourne : Nombre d’employés avec statut ‘billable’ ou ‘grace_period’
  • Exclut : Employés ‘terminated’ et ‘non_billable’
  • Utilisation : Calculs de facturation temps réel

2. calculate_period_cost_with_taxes(license_uuid, period_start, period_end)

Objectif : Calculer le coût total d’une période avec taxes et avenants

Description :

  • Paramètres : UUID licence, dates début/fin de période
  • Calcule : Coût de base + avenants + taxes applicables
  • Retourne : Objet JSON avec montants USD et devise locale
  • Gère : Conversion automatique de devise et application taxes

3. detect_fraud_patterns(tenant_uuid, days_lookback)

Objectif : Analyser les patterns de comportement suspects d’un tenant

Description :

  • Paramètres : UUID tenant, nombre de jours d’analyse
  • Analyse : Variations d’effectif, patterns de congés, timing vs renouvellements
  • Retourne : Score de risque (0-100) avec détails des anomalies
  • Utilisation : Dashboard admin et alertes automatiques

4. get_exchange_rate(from_currency, to_currency, at_date)

Objectif : Récupérer le taux de change historique pour une date donnée

Description :

  • Paramètres : Code devise source, cible, date
  • Recherche : Taux valide à la date demandée
  • Fallback : Taux courant si pas d’historique pour la date
  • Retourne : Taux de change avec métadonnées (date d’application, source)

5. calculate_tax_amount(base_amount_usd, tenant_uuid)

Objectif : Calculer les taxes applicables selon le tenant et son pays

Description :

  • Paramètres : Montant de base en USD, UUID du tenant
  • Vérifie : Statut fiscal du tenant (numéro TVA renseigné?)
  • Applique : Règles fiscales du pays si applicable
  • Retourne : Montant de taxe en USD + détails des règles appliquées

6. generate_billing_summary(license_uuid, billing_cycle_id)

Objectif : Générer un résumé complet de facturation pour affichage

Description :

  • Paramètres : UUID licence et cycle de facturation
  • Compile : Employés facturables, avenants, taxes, historique
  • Formate : Montants dans la devise du tenant avec symboles
  • Retourne : JSON structuré prêt pour interface utilisateur
